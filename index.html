Plan guidance

Keep the main planning inputs simple here. More technical controls now sit in Advanced.

What belongs in Plan

Retirement horizon, initial assets, spending target, asset mix, state pensions, and the spending rule. This should stay focused on the household plan rather than capital-markets detail.

What moved out

Return assumptions, volatility, Monte Carlo run count, seed, and display controls are now in Advanced so the planning page stays readable as the simulator grows.

Current engine scope

This version still runs the existing yearly engine and worker, but the UI is now structured to support the next phase: modular forms, clearer state flow, and later a monthly-first model.

Market assumptions

Simulation settings

Total household spending starts from the initial withdrawal amount and normally rises with inflation. If the inflation-skip rule is enabled, inflation increases are skipped after negative return years. State pension starts at the chosen pension age for each person and reduces the amount drawn from the portfolio. Withdrawals are taken from cashlike assets first, then bonds, then equities.

Advanced roadmap

This panel is now where the next credibility and realism layers belong.

P0 next

Monthly cashflow engine, fuller Guyton-Klinger rules, better inflation handling, and correlation-aware return modelling fit naturally here without cluttering Plan.

P1 later

Tax wrappers, fees, DB pension, annuities, inheritance flows, and glidepaths should each become their own advanced blocks once the engine structure is ready.

Why this matters

The simulator now has a clear place for technical controls. That makes the next refactor simpler: splitting form logic out of app.js without mixing core plan inputs and engine parameters.
